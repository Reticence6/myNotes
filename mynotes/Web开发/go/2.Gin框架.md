# Gin

## 介绍

Gin是一个Go编写的轻量级http web框架， 运行速度非常快

Gin擅长Api接口的高并发

## 环境搭建

1. 创建项目并初始化Go模块

   ```shell
   mkdir gin-quickstart && cd gin-quickstart
   go mod init gin-quickstart
   ```

2. 下载并安装Gin

   ```go
   go get -u github.com/gin-gonic/gin
   ```

3. 编写代码

   ```go
   package main
   
   import "github.com/gin-gonic/gin"
   
   func main() {
     router := gin.Default()
     router.GET("/ping", func(c *gin.Context) {
       c.JSON(200, gin.H{
         "message": "pong",
       })
     })
     router.Run() // 默认监听 0.0.0.0:8080
   }
   ```

4. 运行API服务，在浏览器中访问 http://localhost:8080/ping 即可

   ```go
   go run main.go
   ```

5. Gin结合net/http使用

   - 如果需要用net/http包中的常量表示相应码，同时导入即可

   ```go
   package main
   
   import (
     "github.com/gin-gonic/gin"
     "net/http"
   )
   
   func main() {
     router := gin.Default()
     router.GET("/ping", func(c *gin.Context) {
       c.JSON(http.StatusOK, gin.H{
         "message": "pong",
       })
     })
     router.Run()
   }
   ```

6. 配置golang程序的热加载

   - 使用工具  http://github.com/gravityblast/fresh

   ```go
   go install github.com/pilu/fresh@latest
   // 之后运行时输入fresh即可
   fresh
   ```

## 响应数据

### c.String()

```go
router.GET("/", func(c *gin.Context) {
	c.String(200, "值: %v", "你好gin!")
})
```

### c.JSON()

```go
type Article struct {
	// 结构体字段小写开头时，会被 Go 标记为「包私有」（仅当前包可访问），
	// 而 Gin 的 JSON 序列化依赖reflect反射包，无法跨包访问私有字段，因此序列化结果为空；
	// 大写开头的字段是「公有」的，能被反射访问，才能正常序列化。
	Title   string `json:"title"`
	Desc    string `json:"desc"`
	Content string `json:"content"`
}

func main() {
	// 创建一个默认的路由引擎
	router := gin.Default()

	// 返回JSON
	router.GET("/json", func(c *gin.Context) {
		c.JSON(http.StatusOK, map[string]interface{}{
			"success": true,
			"msg":     "json",
		})
	})
	router.GET("/json2", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"success": true,
			"msg":     "json2",
		})
	})
	router.GET("/json3", func(c *gin.Context) {
		// 实例化结构体时用指针不是语法强制要求，
		// 但在 Gin（及大部分 Go 开发场景）中是推荐写法
		// 核心优势是「避免拷贝开销」+「支持修改原实例」+「符合 Gin 的序列化优化逻辑」。
		a := &Article{
			Title:   "Title",
			Content: "Content",
			Desc:    "Desc",
		}

		c.JSON(http.StatusOK, a)
	})

	// 启动一个web服务
	router.Run(":8000")
}
```

### c.JSONP

`JSONP`（JSON with Padding）是一种**非官方的跨域数据交互协议**，本质不是新技术，而是利用`<script>`标签不受浏览器同源策略限制的特性，实现跨域获取数据的 “hack 技巧”—— 注意：它和 JSON 无关，只是数据格式用 JSON，核心是 “动态脚本注入 + 回调函数”。

JSONP 是 “前端跨域的权宜之计”，解决了早期 AJAX 跨域的问题，但现代开发中，CORS（后端配置`Access-Control-Allow-Origin`等响应头）是更规范、安全的选择。

```go
// 返回JSONP
// http://127.0.0.1:8000/jsonp?callback=xx
// xx({"title":"Title","desc":"Desc","content":"Content"});
router.GET("/jsonp", func(c *gin.Context) {

  a := &Article{
    Title:   "Title",
    Content: "Content",
    Desc:    "Desc",
  }

  c.JSONP(http.StatusOK, a)
})
```

### c.HTML

必须用`router.LoadHTMLGlob("templates/*")`配置模版文件，且需放在定义路由引擎之后，才能够渲染html文件

```go
// 创建一个默认的路由引擎
router := gin.Default()
// 配置模版文件
router.LoadHTMLGlob("templates/*")

// 加载html，渲染模版
router.GET("/news", func(c *gin.Context) {
  c.HTML(http.StatusOK, "news.html", gin.H{
    "title": "news data",
  })
})
router.GET("/goods", func(c *gin.Context) {
  c.HTML(http.StatusOK, "goods.html", gin.H{
    "title": "goods data",
  })
})
```
