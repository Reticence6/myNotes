# Mac安装

1. 在官网 https://go.dev/dl/，选择pkg下载，安装

2. 配置环境

   - 新建项目目录
   - 打开`~/.zshrc`配置：

   ```shell
   # go
   export GOPATH=/Users/jingmo/Desktop/projects/Go/projects
   export GOBIN=$GOPATH/bin   #GOLANG运行目录一般不可修改
   export PATH=$PATH:$GOBIN 
   ```

   - 执行 `source ~/.zshrc`保存修改
   - 输入`go env`查看配置结果

3. 测试

   - 创建文件 main.go

   - 编写代码

     ```go
     package main
      
     import "fmt"
      
     func main(){
         fmt.Println("Hello Go!")
     }
     ```

   - 执行：`go run main.go`

   - 或者执行：`go build main.go` 和 `./main`

# 基础

## 变量定义

1. 变量定义

   ```go
   package main
   import "fmt"
   
   func main() {
       // 先声明
       var name string
       // 再赋值
       name = "wjk"
       fmt.Println(name)
   
       // 声明并赋值
       var name1 string = "wjk"
       fmt.Println(name1)
       // 省略类型
       var name2 = "wjk"
       fmt.Println(name2)
   
       // 声明并赋值 短声明符号 :=  它只能被用在函数体内，而不可以用于全局变量的声明与赋值
       name3 := "wjk"
       fmt.Println(name3)
   }
   ```

2. 局部变量与全局变量

   - 定义在函数里的都是局部变量，包括main
   - 函数外声明的是全局变量，必须要用var，不能用短声明，并且不用也不会报错

3. 定义多个变量

   ```go
   var a1, a2 = 1, 2
   fmt.Println(a1, a2)
   var (
       s1 string = "1"
       s2 string = "2"
   )
   fmt.Println(s1, s2)
   ```

4. 定义常量， 声明后要赋值

   - 常量中的数据类型只可以是布尔型、数字型（整数型、浮点型和复数）和字符串型。

   ```go
   const version string = "2.0.1"
   ```

   - iota，特殊常量，可以认为是一个可以被编译器修改的常量（常量声明的自增枚举常量）
     - iota 在 const关键字出现时将被重置为 0(const 内部的第一行之前)，const 中每新增一行常量声明将使 iota 计数一次(iota 可理解为 const 语句块中的行索引)。
     - 适用于定义状态码、枚举类型、优先级等有规律的常量
     - iota 可以被用作枚举值：

   ```go
   const (
       a = iota
       b = iota
       c = iota
   )
   ```

   - 第一个 iota 等于 0，每当 iota 在新的一行被使用时，它的值都会自动加 1；所以 a=0, b=1, c=2 可以简写为如下形式：

   ```go
   const (
       a = iota
       b
       c
   )
   ```

   - iota实现枚举

   ```go
   // 定义状态枚举
   type Status int
   
   const (
       StatusPending Status = iota // 0：待处理
       StatusSuccess               // 1：成功
       StatusFailed                // 2：失败
       StatusTimeout               // 3：超时
   )
   
   // 给枚举加字符串描述（最佳实践）
   func (s Status) String() string {
       switch s {
       case StatusPending:
           return "待处理"
       case StatusSuccess:
           return "成功"
       case StatusFailed:
           return "失败"
       case StatusTimeout:
           return "超时"
       default:
           return "未知状态"
       }
   }
   
   func main() {
       var s Status = StatusSuccess
       fmt.Println(s) // 输出：成功（调用String()方法）
       fmt.Println(int(s)) // 输出：1（底层数值）
   }
   ```

## 基本数据类型

基本数据类型有：整数型，浮点型，复数，布尔，字符串

### 整数型

- 默认数字定义类型为int类型

- 带个u就是无符号，只能存正整数

- 后面的数字就是二进制的位数

- uint8还有一个别名 byte， 一个字节=8个bit位

- int类型的大小取决于所使用的平台

- 整数类型细分：

  ```go
  var n1 uint8 = 2 
  var n2 uint16 = 2 
  var n3 uint32 = 2 
  var n4 uint64 = 2 
  var n5 uint = 2 
  var n6 int8 = 2 
  var n7 int16 = 2 
  var n8 int32 = 2 
  var n9 int64 = 2 
  var n10 int = 2
  ```

### 浮点型

GO支持两种浮点型：float32和float64

1. float32 的浮点数的最大范围约为3.4e38，可以使用常量定义：`math.MaxFloat32`
2. float64 的浮点数的最大范围约为 1.8e308，可以使用一个常量定义：`math.MaxFloat64`

如果没有显式声明，则默认是float64

### 字符型

- byte-单字节字符，rune-多字节字符

  ```go
  // ascii里的字符
  var a byte = 'a'
  fmt.Printf("%c %d\n", a, a)
  var a1 uint8 = 97
  fmt.Printf("%c %d\n", a1, a1)
  
  // 多字节字符
  var z rune = '中'
  fmt.Printf("%c %d\n", z, z)
  ```

### 字符串类型

- 字符赋值用单引号，字符串类型用双引号

- 转义字符

  ```go
  fmt.Println("枫枫\t知道")              // 制表符
  fmt.Println("枫枫\n知道")              // 回车
  fmt.Println("\"枫枫\"知道")            // 双引号
  fmt.Println("枫枫\r知道")              // 回到行首
  fmt.Println("C:\\pprof\\main.exe")	  // 反斜杠
  ```

- 多行字符

  ```go
  var s = `今天
  天气
  真好
  `
  fmt.Println(s)
  ```

### 布尔类型

- 只有true和false
- 布尔类型变量的默认值为false
- Go语言不允许将整型强制转换为布尔型
- 布尔型无法参与数值运算，也无法与其他类型进行转换

### 零值问题

- 如果我们给一个**基本数据类型**只声明不赋值

  那么这个变量的值就是对应类型的零值，例如int就是0，bool就是false，字符串就是""

  ```go
  var a1 int
  var a2 float32
  var a3 string
  var a4 bool
  
  fmt.Printf("%#v\n", a1)
  fmt.Printf("%#v\n", a2)
  fmt.Printf("%#v\n", a3)
  fmt.Printf("%#v\n", a4)
  ```

### map

- Go语言中的map(映射、字典)是一种内置的数据结构，它是一个`无序`的key-value对的集合
- map的key必须是基本数据类型，value可以是任意类型
- **map使用之前一定要初始化**

```go
// 声明
var m1 map[string]string
// 初始化1
m1 = make(map[string]string)
// 初始化2
m1 = map[string]string{}
// 设置值
m1["name"] = "枫枫"
fmt.Println(m1)
// 取值
fmt.Println(m1["name"])
// 删除值
delete(m1, "name")
fmt.Println(m1)

// 声明并赋值
var m1 = map[string]string{}
var m2 = make(map[string]string)

var userMap map[int]string = map[int]string{
		1: "ff",
		2: "zs",
		4: "",
	}
```

- 取值
  - 如果只有一个参数接，那这个参数就是值，如果没有，这个值就是类型的零值
  - 如果两个参数接，那第二个参数就是布尔值，表示是否有这个元素

```go
// 声明并赋值
var m1 = map[string]int{
  "age": 21,
}
age1 := m1["age1"] // 取一个不存在的
fmt.Println(age1)
age2, ok := m1["age1"]
fmt.Println(age2, ok)
```

## 函数

函数是一段封装了特定功能的可重用代码块，用于执行特定的任务或计算


- 函数返回值

  ```go
  // 命名返回值

  ```

- 匿名函数

- 闭包

  - 本质是一个函数值+该函数所引用的外部变量环境
    - 一个函数嵌套在另一个函数内部（内层函数）；
    - 内层函数引用了外层函数的局部变量（或参数）；
    - 外层函数返回这个内层函数。
  - 此时内层函数即使脱离了外层函数的执行环境，依然能访问并修改这些外部变量 —— 这些变量会和内层函数 “绑定”，不会被垃圾回收，这就是闭包的核心特性。

  ```go
  // 外层函数：返回一个内层函数
  func counter() func() int {
      // 外层函数的局部变量
      count := 0
      // 内层函数：引用了外层的count变量
      return func() int {
          count++ // 修改外层变量
          return count
      }
  }
  
  func main() {
      // 创建第一个闭包实例
      c1 := counter()
      fmt.Println(c1()) // 输出：1
      fmt.Println(c1()) // 输出：2
  
      // 创建第二个闭包实例（独立的变量环境）
      c2 := counter()
      fmt.Println(c2()) // 输出：1
      fmt.Println(c1()) // 输出：3（c1的count不受c2影响）
  }
  ```

- 值传递和引用传递

  - 值传递：把变量的**副本**传递给函数。函数内部修改的是这个副本，不会影响原变量的值。
  - 引用传递：把变量的**内存地址**传递给函数。函数内部通过地址直接操作原变量的内存空间，修改会直接反映到原变量上。
  - **Go 语言中只有值传递，没有真正的引用传递**！很多人误以为切片、map 是引用传递，其实是误解 —— 它们本质是 “值传递”，只是这个 “值” 是指向底层数据的指针。
  - **Go 只有值传递，指针 / 接口能修改原实例，不是因为 “传递方式变了”，而是因为 “传递的是「能找到原实例的地址」”—— 指针直接传地址，接口封装地址，普通值传的是 “实例本身的拷贝”。如果接口封装的是指针类型实例，那么也能修改原实例，如果封装的是值类型的就不能**

  ```go
  func main() {
      // 值传递  开辟空间复制值过去，地址不一样
      var name = "ww"
      fmt.Printf("%p\n", &name)
      copy(name)
      // 引用传递
      set(&name)
      fmt.Println(name)
  }
  func copy(name string) {
      fmt.Printf("func %p\n", &name)
      name = "111"
  }
  
  func set(name *string) {
      fmt.Printf("set-func %p\n", name)
      // 现在name是内存地址，通过内存地址找值
      *name = "11"
  }
  ```

- init函数

  - `init()`函数是一个特殊的函数，存在以下特性：
    1. 不能被其他函数调用，而是在main函数执行之前，自动被调用
    2. init函数不能作为参数传入
    3. 不能有传入参数和返回值
  - 一个go文件可以有多个init函数，谁在前面谁就先执行
  - 用于做初始化

  ```go
  var db int
  func init() {
      db = 10
      fmt.Println("connect db success")
      fmt.Println("init-1")
  }
  func init() {
      fmt.Println("init-2")
  }
  
  func main() {
      fmt.Println("main", db)
  }
  ```

- defer函数

  - 关键字 defer 用于注册延迟调用
  - 这些调用直到 return 前才被执。因此，可以用来做资源清理
  - 多个defer语句，按先进后出的方式执行，谁离return近谁先执行
  - defer语句中的变量，在defer声明时就决定了

  ```go
  func main() {
      defer func() {
         fmt.Println("defer2")
      }()
      defer func() {
         fmt.Println("defer1")
      }()
      return
  }
  ```

## 结构体

- 结构体

  ```go
  type 结构体名称 struct{
      名称 类型//成员或属性
  }
  ```

  ```go
  // Student 定义结构体
  type Student struct {
    Name string

  }
  
  // PrintInfo 给机构体绑定一个方法
  func (s Student) PrintInfo() {

  func main() {
    s := Student{
      Name: "枫枫",
      Age:  21,
    }

  }
  ```

- 结构体嵌套（类似于继承）

  ```go
  type People struct {
    Time string
  }
  // Student 定义结构体
  type Student struct {
    People
    Name string
    Age  int
  }
  ```

- 结构体指针

  ```go
  func (s *Student) SetName(name string) {

      s1 := Student{Name: "ww", Class: c1}
  ```

- 结构体 tag

  - 这个不写json标签转换为json的话，字段名就是属性的名字
  - 如果再转json的时候，不希望某个字段被转出来，可以写一个 `-`
  - `omitempty` : 空值省略

  ```go
  }
  ```

## 类型转换

用一种数据类型的变量转换为另一种类型的变量

基本格式：

```go
type_name(expression)
```

数值类型转换：

- 整型转换为浮点型：

  ```go
  var sum int = 17
  var count int = 5
  var mean float32
  
  mean = float32(sum) / float32(count)
  ```

- 字符串类型转换：

  ```go
  var str string = "10"
  var num int
  // 字符串转整型
  num, err = strconv.Atoi(str)
  // 整型转字符串
  str = strconv.Itoa(num)
  // 字符串转浮点型
  str1 := "3.14"
  num, err = strconv.ParseFloat(str, 64)
  // 浮点型转字符串
  num = 3.14
  str1 = strconv.FormatFloat(num, 'f', 2, 64)
  ```

接口类型转换：

- 类型断言

  - 用于将接口类型转换为指定类型
  - 语法为： value是接口类型的变量，type/T是要转换成的类型，成功则返回转换后的值和一个布尔值

  ```go
  value.(type)
  或者
  value.(T)
  ```

  - 如：

  ```go
  var i interface{} = "hello, world"
  str, ok := i.(string)
  if ok {
  	fmt.Printf("'%s' is a string\n", str)
  } else {
  	fmt.Println("coversion failed")
  }
  ```

- 类型转换

  - 用于将一个接口类型的值转换为另一个接口类型
  - 语法为： T是目标接口类型，value是要转换的值，必须保证转换的值要和目标接口类型是兼容的，否则会报错

  ```go
  T(value)
  ```

  - 如：

  ```go
  // 定义一个接口 Writer
  type Writer interface{
    Write([]byte) (int, error)
  }
  // 实现Writer接口的结构体 StringWriter
  type StringWriter struct{
    str string
  }
  // 实现Write方法
  func (sw *StringWriter) Write(data []byte) (int, error){
    sw.str += string(data)
    return len(data), nil
  }
  
  func main(){
    // 创建一个StringWriter实例并赋值给Writer接口变量
    var w Writer = &StringWriter{}
    // 将Writer接口类型转换为StringWriter类型
    sw := w.(*StringWriter)
    sw.str = "Hello, world"
    fmt.Println(sw.str)
  }
  ```

  - **Go 只有值传递，指针 / 接口能修改原实例，不是因为 “传递方式变了”，而是因为 “传递的是「能找到原实例的地址」”—— 指针直接传地址，接口封装地址，普通值传的是 “实例本身的拷贝”。如果接口封装的是指针类型实例，那么也能修改原实例，如果封装的是值类型的就不能**

  ```go
  package main
  import (
  	"os"
  )
  
  // Writer 定义数据写入的接口，规定写入操作的方法签名
  type Writer interface {
  	// Write 将字节切片数据写入目标存储，返回写入的字节数和可能的错误
  	Write([]byte) (int, error)
  }
  
  // StringWriter 实现Writer接口的结构体，将数据写入内存字符串中
  type StringWriter struct {
  	str string // 存储写入的字符串数据
  }
  
  // Write 实现Writer接口的Write方法，将字节数据追加到StringWriter的str字段
  // 参数：
  //   data - 待写入的字节切片数据
  // 返回：
  //   int - 成功写入的字节数
  //   error - 写入过程中产生的错误（此处无错误，固定返回nil）
  func (sw *StringWriter) Write(data []byte) (int, error) {
  	sw.str += string(data) // 将字节数据转为字符串并追加到str字段
  	return len(data), nil  // 返回写入的字节长度，无错误
  }
  
  // WriteData 通用的数据写入函数，依赖Writer接口实现通用写入逻辑
  // 参数：
  //   w - 实现了Writer接口的实例，负责具体的写入逻辑
  //   data - 待写入的字符串数据
  // 返回：
  //   error - 写入过程中产生的错误
  func WriteData(w Writer, data string) error {
  	// 将字符串转为字节切片，调用Writer接口的Write方法完成写入
  	_, err := w.Write([]byte(data))
  	return err // 返回写入操作的错误信息
  }
  
  func main() {
  	// 1. 使用StringWriter写入数据（内存字符串存储）
  	sw := &StringWriter{}          // 创建StringWriter实例（指针类型，实现Writer接口）
  	WriteData(sw, "拿铁")          // 调用通用写入函数，底层执行*StringWriter.Write
  	
  	// 2. 使用os.File写入数据（文件存储，os.File原生实现了Writer接口）
  	file, _ := os.Create("coffee.txt") // 创建文件实例（忽略错误仅为示例，生产环境需处理）
  	defer file.Close()                 // 确保函数退出时关闭文件
  	WriteData(file, "美式")            // 调用通用写入函数，底层执行os.File.Write
  }
  ```

## 自定义数据类型

在 Go 语言中，自定义类型指的是使用 type 关键字定义的新类型，它可以是基本类型的别名，也可以是结构体、函数等组合而成的新类型。自定义类型可以帮助我们更好地抽象和封装数据，让代码更加易读、易懂、易维护

- 自定义类型

  ```go
  type Code int
  
  const (
    SuccessCode    Code = 0
    ValidCode      Code = 7 // 校验失败的错误
    ServiceErrCode Code = 8 // 服务错误
  )
  
  func (c Code) GetMsg() string {
    // 可能会有更加响应码返回不同消息内容的要求，我们在这个函数里面去实现即可
    // 可能还会有国际化操作
    return "成功"
  }
  
  func main() {
    fmt.Println(SuccessCode.GetMsg())
    var i int
    fmt.Println(int(SuccessCode) == i) // 必须要转成原始类型才能判断
  }
  ```

- 类型别名

  - 和自定义类型很像，但是有一些地方和自定义类型有很大差异
  - 不能绑定方法
  - 打印类型还是原始类型
  - 和原始类型比较，类型别名不用转换

  ```go
  type AliasCode = int
  type MyCode int
  
  const (
    SuccessCode      MyCode    = 0
    SuccessAliasCode AliasCode = 0
  )
  
  // MyCodeMethod 自定义类型可以绑定自定义方法
  func (m MyCode) MyCodeMethod() {
  
  }
  // MyAliasCodeMethod 类型别名 不可以绑定方法
  func (m AliasCode) MyAliasCodeMethod() {
  
  }
  
  func main() {
    // 类型别名，打印它的类型还是原始类型
    fmt.Printf("%T %T \n", SuccessCode, SuccessAliasCode) // main.MyCode int

    fmt.Println(int(SuccessCode) == i) // 必须转换之后才能和原始类型比较
  }
  ```

## 接口

Go 中的接口是**一组方法签名的集合**，它定义了 “做什么”，但不关心 “怎么做”（即不实现具体逻辑）。简单来说：

- 接口是一种**抽象类型**，只声明方法，不包含字段，也不实现方法；

- 任何类型，只要实现了接口中**所有**的方法，就 “隐式” 满足该接口（无需显式声明，这是 Go 的核心特点）；

- 接口变量可以存储任何实现了该接口的类型实例，实现多态。

  ```go
  package main
  
  import "fmt"
  
  // 1. 定义接口：Speaker（能“说话”的事物）
  type Speaker interface {
      Speak() string // 声明方法：返回字符串
  }
  
  // 2. 定义结构体：Dog（狗）
  type Dog struct {
      Name string
  }
  
  // 3. Dog实现Speaker接口的Speak方法（只要实现所有方法，就隐式满足接口）
  func (d Dog) Speak() string {
      return d.Name + "：汪汪汪"
  }
  
  // 4. 定义结构体：Cat（猫）
  type Cat struct {
      Name string
  }
  
  // 5. Cat实现Speaker接口的Speak方法
  func (c Cat) Speak() string {
      return c.Name + "：喵喵喵"
  }
  
  // 6. 接收Speaker接口类型的参数（多态的核心：不关心具体类型，只关心是否有Speak方法）
  func makeSpeak(s Speaker) {
      fmt.Println(s.Speak())
  }
  
  func main() {
      // 实例化Dog和Cat
      dog := Dog{Name: "旺财"} 
      cat := Cat{Name: "小花"}
  
      // 接口变量可以存储任意实现了该接口的类型实例
      var s Speaker
      s = dog // s存储Dog实例
      makeSpeak(s) // 输出：旺财：汪汪汪
  
      s = cat // s存储Cat实例
      makeSpeak(s) // 输出：小花：喵喵喵
  
      // 也可以直接传实例（Go自动隐式转换为接口类型）
      makeSpeak(dog)
      makeSpeak(cat)
  }
  ```

- 类型断言

  - 接口变量存储的是 “类型 + 值”，类型断言用于从接口变量中提取具体类型的值

  ```go
  // 基于Speaker接口的类型断言逻辑
  func checkAndCallSpecificMethod(s Speaker) {
  	// 类型断言：判断接口变量s是否存储的是Dog类型
  	// 格式：具体类型值, 断言是否成功 = 接口变量.(具体类型)
  	dog, ok := s.(Dog)
  	if ok {
  		// 断言成功：调用Dog的专属方法
  		dog.ShowBreed()
  		return
  	}
  
  	// 类型断言：判断接口变量s是否存储的是Cat类型
  	cat, ok := s.(Cat)
  	if ok {
  		// 断言成功：调用Cat的专属方法
  		cat.ShowColor()
  		return
  	}
  
  	// 断言失败：既不是Dog也不是Cat
  	fmt.Println("未知的Speaker类型")
  }
  ```

- 类型开关

  - 批量判断接口变量的具体类型，比多次类型断言更简洁

  ```go
  // 改用类型开关替代多次类型断言
  func checkAndCallSpecificMethod(s Speaker) {
  	// 类型开关：一次性判断接口变量的具体类型
  	switch t := s.(type) {
  	case Dog:
  		// t是Dog类型的实例，可调用专属方法
  		t.ShowBreed()
  	case Cat:
  		// t是Cat类型的实例，可调用专属方法
  		t.ShowColor()
  	case Bird:
  		// 新增对Bird的处理
  		fmt.Printf("%s是一只小鸟，只会叽叽叫\n", t.Name)
  	default:
  		fmt.Println("未知的Speaker类型")
  	}
  }
  ```

- 空接口

  - 空接口可以接受任何类型，即任何类型都实现了空接口的定义

## 协程和channel

- 协程

  - 协程（Goroutine，简称 Goroutine）是 Go 语言**轻量级的用户态线程**，由 Go 运行时（runtime）而非操作系统内核管理。
  - `main`函数运行在主线程（main goroutine）中，程序的生命周期由 main goroutine 决定；一旦 main goroutine 退出，无论其他协程是否执行完成，整个程序都会终止。
  - 同一个进程的所有协程共享进程的内存空间，因此协程间可以直接访问同一个变量（但需要注意并发安全）。
  - 用 sync.WaitGroup 等待多个协程

  ```go
  func shopping(name string, money int, wait *sync.WaitGroup) {
      fmt.Printf("%s shopping\n", name)
      time.Sleep(1 * time.Second)
      fmt.Printf("%s stop\n", name)
  
      wait.Done()
  }
  
  func main() {
      var wait sync.WaitGroup
  
      startTime := time.Now()
      // 协程
      wait.Add(3)
      // 主线程结束，协程函数跟着结束
      go shopping("zhangsan", 2, &wait)
      go shopping("lisi", 3, &wait)
      go shopping("wangwu", 4, &wait)
  
      wait.Wait()
      fmt.Println("success", time.Since(startTime))
  }
  ```

- channel

  - 协程间直接共享变量容易引发竞态条件（Race Condition），Go 推荐使用**通道（Channel）** 实现协程间的安全通信（而非共享内存）。

  ```go
  // 声明并初始化一个长度为0的信道 channel
  var moneyChan = make(chan int)
  
  func shopping(name string, money int, wait *sync.WaitGroup) {
      fmt.Printf("%s shopping\n", name)
      time.Sleep(1 * time.Second)
      fmt.Printf("%s stop\n", name)
  
      moneyChan <- money
  
      wait.Done()
  }
  
  func main() {
      var wait sync.WaitGroup
  
      startTime := time.Now()
  
      // 协程
      wait.Add(3)
      // 主线程结束，协程函数跟着结束
      go shopping("zhangsan", 2, &wait)
      go shopping("lisi", 3, &wait)
      go shopping("wangwu", 4, &wait)
  
    	// 启动一个单独的协程：等待所有购物协程完成后，关闭Channel
      go func() {
         wait.Wait()
         close(moneyChan)
      }()
  
      for {
         money, ok := <-moneyChan
         fmt.Println(money, ok)
         if !ok {
            break
         }
      }
  
      fmt.Println("success", time.Since(startTime))
  }
  ```

- select

  - select是 Go 语言专门为 **Channel 通信** 设计的关键字，作用是：**同时监听多个 Channel 的读写操作，一旦其中一个 Channel 可以操作（可读 / 可写），就执行对应的分支逻辑**。
  - 核心特性：
    - **多路监听**：一次可以监听多个 Channel，解决 “同时处理多个 Channel 数据” 的问题；
    - **非公平性**：如果多个 Channel 同时就绪，Go 会随机选择一个分支执行（避免某一个 Channel 被 “饿死”）；
    - **阻塞特性**：如果所有 Channel 都未就绪，`select` 会阻塞，直到有一个 Channel 就绪；
    - **default 分支**：可选，若所有 Channel 未就绪，会立即执行 default 分支（非阻塞）。
  - 退出信号
    - `doneChan` 是一个 **“退出信号通道”**（用 `struct{}` 是因为它占用 0 内存，仅做信号传递）
    - 当所有协程完成后，关闭 `doneChan`
    - `select` 监听到 `doneChan` 关闭（`<-doneChan` 会立即返回），执行 `return` 退出循环，避免程序卡死；
    - 它是比 “仅靠 Channel 关闭” 更可靠的 “退出触发器”—— 因为有多个 Channel 时，无法通过单个 Channel 的关闭判断 “所有数据都已接收”。

  ```go
  // 声明并初始化一个长度为0的信道 channel
  var moneyChan1 = make(chan int)
  var nameChan1 = make(chan string)
  
  // 声明一个用于关闭的信道
  var doneChan = make(chan struct{})
  
  // var wait sync.WaitGroup
  
  func pay(name string, money int, wait *sync.WaitGroup) {
      fmt.Printf("%s shopping\n", name)
      time.Sleep(1 * time.Second)
      fmt.Printf("%s stop\n", name)
  
      moneyChan1 <- money
      nameChan1 <- name
  
      wait.Done()
  }
  
  func main() {
      var wait sync.WaitGroup
      startTime := time.Now()
  
      // 协程
      wait.Add(3)
      // 主线程结束，协程函数跟着结束
      go pay("zhangsan", 2, &wait)
      go pay("lisi", 3, &wait)
      go pay("wangwu", 4, &wait)
  
      go func() {
         defer close(moneyChan1)
         defer close(nameChan1)
         defer close(doneChan)
         wait.Wait()
      }()
  
      var moneyList []int
      var nameList []string
      var event = func() {
         for {
            select {
            case money := <-moneyChan1:
               moneyList = append(moneyList, money)
            case name := <-nameChan1:
               nameList = append(nameList, name)
            // doneChan 是一个 “退出信号通道”（用 struct{} 是因为它占用 0 内存，仅做信号传递）
            case <-doneChan:
               return
            }
         }
      }
      event()
      fmt.Println("success", time.Since(startTime))
      fmt.Println(moneyList)
      fmt.Println(nameList)
  }
  ```

- 协程超时处理

  - **给协程的执行设置一个 “时间上限”，如果在规定时间内协程未完成，就触发超时逻辑（比如终止任务、返回默认值、打印日志等）**。
  - 核心工具：
    - `time.After(duration)`：返回一个 Channel，在指定时长后会向该 Channel 发送一个时间值
    - `select`：监听 “协程完成信号 Channel” 和 “超时信号 Channel”，哪个先就绪就执行哪个分支。

  ```go
  var done = make(chan struct{})
  
  func event() {
      fmt.Println("event begin")
      time.Sleep(2 * time.Second)
      fmt.Println("event end")
      close(done)
  }
  
  func main() {
      go event()
      select {
      case <-done:
         fmt.Println("协程执行完毕")
      case <-time.After(time.Second):
         fmt.Println("超时")
         return
  
      }
  }
  ```

- 线程安全

  - 使用同步锁解决

  ```go
  var num int
  var wait  sync.WaitGroup
  var lock  sync.Mutex
  
  func add() {
    // 谁先抢到了这把锁，谁就把它锁上，一旦锁上，其他的线程就只能等着
    lock.Lock()
    for i := 0; i < 1000000; i++ {
      num++
    }
    lock.Unlock()
    wait.Done()
  }
  func reduce() {
    lock.Lock()
    for i := 0; i < 1000000; i++ {
      num--
    }
    lock.Unlock()
    wait.Done()
  }
  
  func main() {
    wait.Add(2)
    go add()
    go reduce()
    wait.Wait()
    fmt.Println(num)
  
  }
  ```

  - 线程安全的map
    - 使用`sync.Map{}`

  | 方法                                         | 作用                                                         |
  | :------------------------------------------- | :----------------------------------------------------------- |
  | `Store(key, value)`                          | 存储键值对（等价于普通 map 的`m[key] = value`）              |
  | `Load(key)`                                  | 获取 key 对应的值，返回`(value interface{}, ok bool)`（ok=true 表示存在） |
  | `LoadOrStore(key, value)`                    | 获取 key 的值，若不存在则存储，返回`(actual interface{}, loaded bool)` |
  | `Delete(key)`                                | 删除 key（等价于普通 map 的`delete(m, key)`）                |
  | `Range(f func(key, value interface{}) bool)` | 遍历所有键值对（f 返回 false 时停止遍历）                    |

  ```go
  var wait1 sync.WaitGroup
  // var mp = map[string]string{}
  var mp = sync.Map{}
  var lock1 sync.Mutex
  
  func reader() {
      for {
         /*lock1.Lock()
         fmt.Println(mp["time"])
         lock1.Unlock()*/
         fmt.Println(mp.Load("time"))
      }
      wait1.Done()
  }
  func writer() {
      for {
         /*lock1.Lock()
         mp["time"] = time.Now().Format("15:04:05")
         lock1.Unlock()*/
         mp.Store("time", time.Now().Format("15:04:05"))
      }
      wait1.Done()
  }
  
  func main() {
      wait1.Add(2)
      go reader()
      go writer()
      wait1.Wait()
  }
  ```

## 异常处理

- Go 的异常处理分为两层：

  - 普通错误：用`error`接口返回，调用方显式检查处理；

  - 严重异常：用`panic`触发，`recover`（配合`defer`）捕获，避免程序崩溃；

- 向上抛

  - 将错误交给上一级处理

  ```go
  func div(a, b int) (res int, err error) {
      if b == 0 {
         err = errors.New("除数不能为0")
         return
      }
      res = a / b
      return
  }
  
  func server() (res int, err error) {
      res, err = div(2, 1)
      if err != nil {
         // 把错误向上传递
         return
      }
      // 执行其他逻辑
      res++
      return
  }
  
  func main() {
      res, err := server()
      if err != nil {
         fmt.Println(err)
         return
      }
      fmt.Println(res)
  }
  ```

- 中断程序

  - 遇到错误直接停止，一般用于初始化

  ```go
  func init() {
      _, err := os.ReadFile("1234")
      fmt.Println(err)
      if err != nil {
         // log.Fatalf("错误了")
         panic("错误了")
      }
  }
  
  func main() {
      fmt.Println("main")
  }
  ```

- 恢复程序

  - 使用recover捕获异常

  ```go
  func read() {
      defer func() {
         err := recover()
         if err != nil {
            fmt.Println(err)
            fmt.Println(string(debug.Stack()))
         }
      }()
      var list []int = []int{1, 2}
      fmt.Println(list[2])
  
  }
  func main() {
      read()
      fmt.Println("正常逻辑")
  }
  ```

- 核心规则：

  - `recover`必须在`defer`中调用，且只能捕获当前协程的 panic；
  - 不要滥用 panic/recover，普通错误优先用 error；

- 最佳实践：显式处理 error，在协程 / 顶层函数中用 recover 捕获 panic，避免程序整体崩溃。

## 泛型

- 泛型的核心目的是：**让函数 / 类型支持多种数据类型，避免重复写相似代码**

  | 概念             | 作用                                                 | 示例                                                 |
  | :--------------- | :--------------------------------------------------- | :--------------------------------------------------- |
  | **类型参数**     | 在函数或类型名后声明，表示待定的类型。               | `[T any]`                                            |
  | **类型约束**     | 定义类型参数必须满足的条件（如支持的操作符或方法）。 | `int，float64，comparable，constraints.Ordered，any` |
  | **`any`**        | 约束类型参数为**任何**类型。                         | `[T any]`                                            |
  | **`comparable`** | 约束类型参数为**可比较**的类型。                     | `[K comparable]`                                     |

- 泛型函数：

  ```go
  // 基本语法结构
  func 函数名[T 约束](参数 T) 返回值类型 {
      // 函数体
  }
  
  type 类型名[T 约束] struct {
      // 结构体字段
  }
  ```

- 文件权限

  - 主要用于linux系统，在windows下这个参数会被无视，代表文件的模式和权限位，

  - 三个占位符

    - 第一个是文件所有者所拥有的权限
    - 第二个是文件所在组对其拥有的权限
    - 第三个占位符是指其他人对文件拥有的权限

  - 根据UNIX系统的权限模型，文件或目录的权限模式由三个数字表示，分别代表 所有者（Owner） 、组（Group） 和 其他用户(Other) 的权限。每个数字由三个比特位组成，分别代表读、写和执行权限。因此，对于一个mode参数值，它的每一个数字都是一个八进制数字，代表三个比特位的权限组合

    ```Go
    R：读，Read的缩写，八进制值为 4；
    W：写，Write的缩写，八进制值为 2；
    X：执行，Execute的缩写，八进制值为 1；
    ```

  - 例如

    ```Go
    0444 表示三者均为只读的权限；
    0666 表示三者均为“读写”的权限；
    0777 表示三者均为读写执行的权限；
    0764 表示所有者有读写执行（7=4+2+1）的权限，组有读写（6=4+2）的权限，其他用户则为只读（4=4）；
    ```

### 结构体反射

#### 结构体反射

- 结构体字段**首字母大写**（导出字段）：这是反射能访问字段的前提（小写字段反射无法读取）
- `reflect.ValueOf(obj)`：把普通变量（这里是`Student`实例）转换成反射**值对象**（`reflect.Value`），可以通过它获取 / 修改字段的值；
- `reflect.TypeOf(obj)`：把普通变量转换成反射**类型对象**（`reflect.Type`），可以通过它获取字段名、tag、字段类型等元信息；
- `v.NumField()`：获取结构体字段总数
- `t.Field(i)`：获取字段的类型信息（`reflect.StructField`）
- `tf.Tag.Get("json")`：读取结构体标签
- `v.Field(i)`：获取字段的反射值对象，返回第`i`个字段的`reflect.Value`对象，直接打印时会输出字段的实际值

```go
type Student struct {
    Name  string `json:"name"`
    Age   int
    IsMan bool
}

func ParseJson(obj any) { // any等价于interface{}，接收任意类型参数
	// 步骤1：获取传入参数的反射Value（值信息）和Type（类型信息）
	v := reflect.ValueOf(obj) // 反射值对象：存储结构体的实际值
	t := reflect.TypeOf(obj)  // 反射类型对象：存储结构体的类型信息（字段名、tag、类型等）

	// 步骤2：遍历结构体的所有字段（v.NumField()返回结构体字段总数）
	for i := range v.NumField() { // 等价于 for i := 0; i < v.NumField(); i++
		// 步骤3：获取第i个字段的类型信息（Type）
		tf := t.Field(i) // tf是reflect.StructField类型，包含字段名、tag、类型等

		// 步骤4：读取字段的json tag
		jsonTag := tf.Tag.Get("json") // 从tag中提取"json"对应的值
		if jsonTag == "" {            // 如果没有json tag，用字段名替代
			jsonTag = tf.Name
		}

		// 步骤5：输出json tag（或字段名）和字段值
		fmt.Println(jsonTag, v.Field(i)) // v.Field(i)获取第i个字段的反射值对象
	}
}
func main() {
    s := Student{Name: "ww", Age: 20, IsMan: true}
    ParseJson(s)
}
```

#### 通过反射修改结构体的值

-  必须传入**结构体指针**（而非结构体值）
- 必须通过`.Elem()`解引用指针，将 “指针类型的反射值” 转为 “结构体类型的反射值”

```go
type User struct {
    Name1 string `big:"-"`
    Name2 string
}
func SetStruct(obj any) { // any = interface{}，接收任意类型（这里必须传结构体指针）
	// 步骤1：获取反射值对象 + 解引用指针（关键！修改值的前提）
	v := reflect.ValueOf(obj).Elem() 
	// 解释：
	// - reflect.ValueOf(obj)：传入的是&s（User指针），所以v初始是"指针类型的反射值"；
	// - .Elem()：解引用指针，得到结构体本身的反射值（Kind=struct），只有这样才能修改字段。

	// 步骤2：获取反射类型对象 + 解引用指针（和值对象对应）
	t := reflect.TypeOf(obj).Elem()
	// 解释：t最终是User结构体的类型对象，用于读取字段标签。

	// 步骤3：遍历结构体所有字段
	for i := 0; i < v.NumField(); i++ {
		// 步骤4：获取第i个字段的反射值对象（可修改）
		value := v.Field(i) // value是Name1/Name2字段的反射值对象

		// 步骤5：读取字段的big标签
		big := t.Field(i).Tag.Get("big") 
		// 解释：
		// - t.Field(i)：获取第i个字段的类型信息（StructField）；
		// - .Tag.Get("big")：读取big标签的值，Name1返回"-"，Name2返回""。

		// 步骤6：若big标签为空，跳过该字段
		if big == "" {
			continue // Name2的big标签为空，直接跳过，不修改
		}

		// 步骤7：修改字段值（转为大写后赋值）
		value.SetString(strings.ToUpper(value.String()))
		// 解释：
		// - value.String()：获取字段的字符串值（Name1初始是"name1"）；
		// - strings.ToUpper()：转为大写（"NAME1"）；
		// - value.SetString()：将大写值赋值给字段（修改原结构体的字段值）。
	}
}

func main() {
    s := User{Name1: "name1", Name2: "name2"}
    SetStruct(&s)
    fmt.Println(s)
}
```

#### 调用结构体方法

```go
type User1 struct {
}

func (User1) Call(name string) {
    fmt.Println("call---", name)
}

func Call(obj any) { // any = interface{}，接收任意类型（这里必须传结构体指针）
	// 步骤1：获取反射值对象 + 解引用指针
	v := reflect.ValueOf(obj).Elem() 
	// 解释：
	// - 传入的是&s（User1指针），reflect.ValueOf(obj)得到“指针类型的反射值”；
	// - .Elem()解引用，得到“User1值类型的反射值”（匹配方法的值接收者）；
	// - 若方法是指针接收者，无需.Elem()，直接用v即可。

	// 步骤2：获取反射类型对象 + 解引用指针
	t := reflect.TypeOf(obj).Elem()
	// 解释：t最终是User1结构体的类型对象，用于读取方法名。

	// 步骤3：遍历结构体的所有方法（v.NumMethod()返回方法总数）
	for i := 0; i < v.NumMethod(); i++ {
		// 步骤4：获取第i个方法的类型信息（Method）
		m := t.Method(i) 
		// 解释：m是reflect.Method类型，包含方法名（m.Name）、方法类型（m.Type）等。

		// 步骤5：筛选出名为"Call"的方法
		if m.Name != "Call" {
			continue // 非Call方法直接跳过
		}

		// 步骤6：获取第i个方法的反射值对象（可调用的方法对象）
		method := v.Method(i) 
		// 解释：method是reflect.Value类型，代表“可执行的方法”。

		// 步骤7：构造方法参数（[]reflect.Value类型）
		// reflect.ValueOf("ww")：将普通字符串转为反射值对象（匹配方法的string参数）
		args := []reflect.Value{reflect.ValueOf("ww")}

		// 步骤8：调用方法并传入参数
		method.Call(args)
		// 解释：
		// - Call方法接收[]reflect.Value类型的参数，返回[]reflect.Value类型的返回值；
		// - 这里方法无返回值，所以无需处理返回结果。
	}
}

func main() {
```

## 网络编程

### TCP

服务端

```go
func main() {
    addr, _ := net.ResolveTCPAddr("tcp", "127.0.0.1:8080")
    listener, err := net.ListenTCP("tcp", addr)
    if err != nil {
       fmt.Println(err)
       return
    }
    fmt.Println("tcp server bind addr: ", addr.String())
    for {
       conn, err := listener.Accept()
       if err != nil {
          break
       }
       fmt.Println(conn.RemoteAddr())
       conn.Write([]byte("hello world"))
       time.Sleep(2 * time.Second)
       conn.Close()
    }
}
```

客户端

```go
func main() {
    conn, err := net.Dial("tcp", "127.0.0.1:8080")

       }

    }
}
```

## 部署

go项目的部署特别简单，编写完成之后，只需要执行go build即可打包为可执行文件

> 注意，这个操作是不同平台不一样的

windows下打包就是exe文件，linux下打包就是二进制文件

### 打包命令

```Go
go build
```

打当前目录下的main包，注意，只能有一个main函数的包

```Go
go build xxx.go
```

打当前目录下，xxx.go的包，这个包必须得是一个main包，不然没有效果

```Go
go build -o main.exe xxx.go
```

强制对输出的文件进行重命名

-o参数必须得在文件的前面-

### 交叉编译

什么是交叉编译呢，就是在windows上，我开发的go程序，我也能打包为linux上的可执行程序

例如在windows平台，打linux的包

> 注意，执行set这个命令，一定得要是在cmd的命令行下，powershell是无效的

```Go
set CGO_ENABLED=0  
set GOOS=linux  
set GOARCH=amd64  
go build -o main main.go
```

CGO_ENABLED : CGO 表示 golang 中的工具，CGO_ENABLED=0 表示 CGO 禁用，交叉编译中不能使用 CGO
GOOS : 环境变量用于指定目标操作系统，mac 对应 darwin，linux 对应 linux，windows 对应 windows ，还有其它的 freebsd、android 等

`GOARCH`：环境变量用于指定处理器的类型，386 也称 `x86` 对应 32位操作系统、`amd64` 也称 x64 对应 64 位操作系统，`arm` 这种架构一般用于嵌入式开发。比如 `Android` ， `iOS` ， `Win mobile` 等

为了方便呢，可以在项目的根目录下写一个bat文件

这样就能快速构建了

然后放到linux服务器下，设置文件权限就可以直接运行了

```Go
chmod +x main
./main
```
